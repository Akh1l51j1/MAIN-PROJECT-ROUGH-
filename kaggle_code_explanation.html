Helper Functions Explained

image_to_base64(img)
Converts an OpenCV image (numpy array) to a Base64 string. Used to send images over HTTP as JSON text instead of binary files.
forensic_upscale(img)
Makes tiny license plates readable:
1. Resize 3× using Lanczos interpolation (highest quality upscale)
2. Convert to grayscale — OCR works better without color
3. Apply CLAHE — enhances contrast to make faded text pop
is_blurry(gray, threshold=80)
Checks if an image is too blurry for OCR using the Laplacian variance method. Sharp images have high variance (lots of edges). Blurry images have low variance. If variance < 80, skip it.
clean_text(text)
Strips all non-alphanumeric characters and converts to uppercase. Turns "JV-316 S" → "JV316S".
similar(a, b)
Fuzzy string matching using SequenceMatcher. Returns 0.0–1.0 similarity ratio. Example: similar("JV316S", "JV3165") → 0.83 (83% match). Used because OCR often confuses S↔5, O↔0, etc.
get_video_fps(path)
Gets the frames-per-second of a video. Used to calculate timestamps and how many frames = 4 seconds.
center(box) / distance(p1, p2) / iou(box1, box2)
Geometry helpers. center finds the midpoint of a bounding box. distance measures pixel distance between two points. iou (Intersection over Union) measures how much two boxes overlap — used in tracking.
extract_impact_centers(frame)
Finds the center coordinates of all cars in the crash frame. These points define the "impact zone" — only cars near these points are checked for plates.

scan_camera_feed() — The Tracking Worker

This is the core tracking function. It runs in a separate thread (one per camera) and scans an entire video looking for a specific license plate.

Open video → Read frames one by one
▼
Run YOLOv8 tracking → Find all cars with IDs
▼
Crop bottom 50% of each car (bumper/plate area)
▼
Forensic upscale → Run OCR → Clean text
▼
Compare with target plate (fuzzy match ≥ 80%)
▼
Match found? → Save snapshot + timestamp → Stop

💡 Each camera thread creates its own YOLO model and OCR reader instance to avoid thread-safety issues. The thread_lock is used when loading the model.

detect_and_callback() — Deep Dive

Phase 1: Incident Detection

# Sliding window approach to reduce false positives
recent_accidents = deque(maxlen=30)  # Last 30 frames
recent_fights = deque(maxlen=30)

while cap.isOpened():
    ret, frame = cap.read()  # Read one frame

    # Run custom YOLO model
    results = custom_model.predict(frame, conf=0.60)

    # Check what was detected in this frame
    is_acc = 'accident' in frame_classes
    recent_accidents.append(1 if is_acc else 0)

    # If 15+ of last 30 frames had accidents → CONFIRMED
    if sum(recent_accidents) >= 15:
        incident_type = 'accident'
        break  # Stop scanning

💡 Why a sliding window? A single frame might be a false positive (motion blur, weird angle). By requiring 15 out of 30 consecutive frames, we only trigger on sustained, real incidents.

Phase 2: Plate Extraction

Phase 2 — Plate Voting Algorithm

Since OCR gives slightly different readings each frame, we use a voting system:

  • Ideal case: Find plates with exactly 3 letters + 3 digits → Pick the most frequent one
  • Fallback: Group plates by first 3 characters → Score by letter count + occurrence frequency → Pick the highest-scored

Phase 3: Cross-Camera Tracking

# For each detected plate, search all cameras in parallel
for plate in detected_plates:
    threads = []

        thread = Thread(target=scan_camera_feed, args=(cam_id, cam_info, plate))
        threads.append(thread)
        thread.start()
