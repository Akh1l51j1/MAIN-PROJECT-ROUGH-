🚨 Road Surveillance System

Complete team study guide — How the system works, technologies used, custom AI model, and the full request flow explained.

📋 1. What Does This System Do?

Our system is an AI-powered road surveillance platform that:

  1. Detects anomalies — Automatically identifies traffic accidents and street fights from surveillance camera video feeds
  2. Reads license plates — When an accident is detected, it extracts the license plates of the vehicles involved using OCR
  3. Tracks vehicles — Searches across multiple camera feeds to find where a specific vehicle has been spotted
  4. Dashboard — Displays all incidents in a real-time web dashboard with snapshots, timelines, and vehicle routes

🏗️ 2. Architecture Overview

The system consists of two servers running on different machines, connected via ngrok tunnels:

🐍 Django Server (Your PC)

Web dashboard, user authentication, database (SQLite). Runs on localhost:8000. Exposed via ngrok.

⚗️ Flask Server (Kaggle GPU)

AI processing — YOLO detection, EasyOCR, vehicle tracking. Runs on port 5001. Exposed via ngrok.

🤖 4. The Custom YOLO Model

What is YOLO?

YOLO (You Only Look Once) is a real-time object detection algorithm. It looks at an entire image in a single pass and predicts:

  • What objects are in the image (class: accident, fight, car, etc.)
  • Where they are (bounding box coordinates)
  • How confident the model is (0–100%)

Our Custom Model (best.pt)

We fine-tuned a pre-trained YOLOv8 model to detect two specific classes:

ClassDescription
accidentVehicle collision / crash scenes
fightStreet fights / physical altercations

How the Model Was Trained

▼

🔬 5. The 3-Phase Detection Pipeline

When you click "Run Accident Detection", here's what happens:

PHASE 1

Incident Detection

The custom YOLO model scans each video frame and classifies it as accident, fight, or normal.

Uses a sliding window of 30 frames — an incident is confirmed only when 15+ out of 30 consecutive frames are positive. This prevents false alarms from a single noisy frame.

ParameterValueMeaning
CONFIDENCE_THRESHOLD0.60Minimum 60% confidence to count a detection
DETECTION_WINDOW_SIZE30Rolling window of 30 frames
ACTIVATION_THRESHOLD15Need 15 positive frames to confirm
PHASE 2

License Plate Extraction (accidents only)

Goes back 4 seconds before the crash, finds cars near the impact zone, crops the bumper area, and runs OCR.

Go back 4 sec before crash
▼
Detect cars near impact zone (within 120px radius)
▼
Crop bottom 50% of car (bumper/plate area)
▼
Forensic upscale 3× + CLAHE contrast enhancement
▼
Skip blurry crops (Laplacian variance check)
▼
Run EasyOCR → Vote for most common plate text
PHASE 3

Cross-Camera Vehicle Tracking

For each detected plate, spawns one parallel thread per camera to search all feeds simultaneously. Uses fuzzy string matching (80% similarity threshold) to handle partial OCR reads.

🔄 6. The Async Callback Pattern

AI processing takes 1–5 minutes, but web requests time out in 30 seconds. The solution:

👤 User
Clicks "Run Accident Detection"
🐍 Django
Creates IncidentLog (is_processed = false)
🐍 Django
Sends request to Kaggle in background thread
🐍 Django
Redirects user to loading page immediately
👤 Browser
⏳ Loading page polls /api/check-incident-status/ every 2 seconds
⚗️ Kaggle
Processing video on GPU... (1–5 minutes)
⚗️ Kaggle
POSTs results to /api/incident/create/
🐍 Django
Saves Incident + Vehicles to database, sets is_processed = true
👤 Browser
✅ Next poll sees is_processed = true → Auto-redirects to dashboard

🧠 10. Key Concepts to Understand

ngrok — Making local servers accessible

ngrok creates a secure tunnel from the internet to your local machine. When you run ngrok http 8000, it gives you a public URL like https://xxx.ngrok-free.dev that forwards traffic to your localhost:8000.

CSRF Exemption

The callback APIs use @csrf_exempt because they receive POST requests from Kaggle (an external server), not from a browser with CSRF tokens.

Background Threading

Django uses threading.Thread to send requests to Kaggle without blocking the HTTP response. The thread is marked daemon=True so it auto-stops when Django shuts down.

Forensic Upscaling

License plates in surveillance footage are tiny. The system: crops the bumper area → upscales 3× with Lanczos interpolation → converts to grayscale → applies CLAHE (Contrast Limited Adaptive Histogram Equalization).

Fuzzy String Matching

OCR is imperfect — 845ZD5 vs 845ZDS. The system uses SequenceMatcher with 80% similarity threshold to accept partial matches.

❓ 12. Common Questions

Q: Why not run everything on one server?

The YOLO model needs a GPU. Kaggle provides free GPUs. Your PC likely doesn't have a powerful enough GPU.

Q: Why does the ngrok URL change?

Free ngrok accounts get a random URL each time. Paid accounts can get a fixed domain.

Q: What happens if Kaggle times out?

Kaggle notebooks auto-stop after ~12 hours. You need to re-run the cells. Django handles timeouts gracefully — if Kaggle doesn't respond, it marks logs as processed so the UI doesn't hang forever.

Q: Can we add more cameras?

Yes! Update the CAMERA_FEEDS dictionary in the Kaggle notebook with more video paths. The tracking system automatically spawns one thread per camera.

Q: How accurate is the model?

It depends on the training data quality. The sliding window (15/30 threshold) reduces false positives. You can tune CONFIDENCE_THRESHOLD, DETECTION_WINDOW_SIZE, and ACTIVATION_THRESHOLD.
